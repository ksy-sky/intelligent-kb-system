# База знаний интеллектуальной обучающей системы по дисциплине «ПиОИвИС»

Данный проект представляет собой реализацию базы знаний для интеллектуальной обучающей системы по дисциплине **«Представление и обработка информации в интеллектуальных системах» (ПиОИвИС)**. База знаний построена на онтологическом формализме с использованием JSON-файлов и предоставляет три ключевых сценария использования: изучение теоретического материала, работа с глоссарием и взаимодействие с интеллектуальным ассистентом на базе LLM.

##  Архитектура проекта

Проект организован по модульному принципу. Центральным компонентом является загрузчик `KnowledgeBaseLoader`, который читает JSON-файлы, валидирует их и строит in-memory индексы для быстрого доступа.

```
├── data/                          # Хранилище онтологических JSON-файлов
│   ├── glossary_T01.json ...      # Теоретические модули (Т01–Т07)
│   └── glossary_labs_t1.json ...  # Лабораторные модули (L01–L05)
├── src/
│   ├── kb_loader.py               # Загрузчик, индексация, публичное API
│   ├── kb_validator.py            # Валидация схемы, ссылок, ацикличности
│   ├── scenario_study.py          # Сценарий 1 (навигация по разделам)
│   ├── scenario_glossary.py       # Сценарий 2 (точный/нечёткий поиск терминов)
│   └── scenario_assistant.py      # Сценарий 3 (LLM-ассистент через Ollama)
├── main.py                        # Точка входа с главным меню
└── README.md
```

**Ключевые модули**:

- **`kb_loader.py`**: строит индексы `concepts_by_id` (O(1) доступ) и `concepts_by_term` (регистронезависимый поиск).
- **`kb_validator.py`**: проверяет форматы ID (C001, CL005), целостность ссылок и отсутствие циклов в иерархии (`надкласс`).
- **`scenario_assistant.py`**: интегрируется с локальным Ollama (модель `llama3.2`) для генерации ответов на естественном языке.

##  Функциональные сценарии

Все сценарии доступны через главное меню (`python main.py`).

### Сценарий 1: Изучение теоретического материала

- **Назначение**: самостоятельная навигация по иерархии понятий.
- **Команды**:
  - Ввод `T01` … `T07` или `L01` … `L05` → список всех понятий раздела.
  - Ввод термина (например, `онтология`) или ID концепта (`C158`) → карточка понятия (определение, связи, примеры).
- **Пример**:
  ```
  Запрос: T07
  > Раздел T07: Технология Semantic Web и онтологии
    Всего понятий: 24
    - RDF
    - OWL
    - SPARQL
    ...
  ```

### Сценарий 2: Глоссарий

- **Назначение**: быстрый поиск определений с поддержкой опечаток.
- **Особенности**:
  - Точный поиск по термину или ID.
  - Нечёткий поиск через `difflib` (при отсутствии точного совпадения).
  - Разрешение межпонятийных связей (замена ID на термины).
- **Пример**:
  ```
  Термин: квантор
  > Похожие термины: квантификатор, квантор существования
  ```

### Сценарий 3: Интеллектуальный ассистент

- **Назначение**: ответы на вопросы естественного языка с использованием LLM и контекста базы знаний.
- **Логика**:
  1. Вопрос анализируется (определяется тип: определение, сравнение, перечисление и т.д.).
  2. Из базы извлекаются релевантные концепты (по ключевым словам).
  3. Формируется промпт с контекстом → запрос к LLM → ответ JSON.
- **Пример**:
  ```
  Вопрос: Чем отличается информация от данных?
  > Информация — это осмысленные данные, обладающие ценностью для принятия решения, тогда как данные — это сырые факты и символы без интерпретации.
  ```

##  Структура базы знаний (JSON)

Каждый JSON-файл содержит массив `topics`. Один топик (`topic`) включает массив `concepts`.

### Формат одного концепта

```json
{
  "concept_id": "C042",
  "term": "онтология",
  "topic_id": "T07",
  "definition": "Явная спецификация концептуализации",
  "examples": ["Ontology Web Language (OWL)", "Protégé Editor"],
  "relations": {
    "надкласс": ["C040"],
    "используется в": ["Semantic Web", "Интеграция данных"],
    "разбиение": ["Таксономии", "Аксиомы", "Индивиды"]
  }
}
```

**Поля**:

| Поле           | Обязательное | Тип                           | Описание                                      |
|----------------|--------------|-------------------------------|-----------------------------------------------|
| `concept_id`   | ✅           | `string` (Cxxx / CLxxx)       | Глобальный уникальный идентификатор.          |
| `term`         | ✅           | `string`                      | Название понятия.                             |
| `topic_id`     | ✅           | `string` (Txx / Lxx)          | Принадлежность к разделу.                     |
| `definition`   | ❌           | `string`                      | Текстовое определение.                        |
| `examples`     | ❌           | `array[string]`               | Примеры использования.                        |
| `relations`    | ❌           | `object`                      | Семантические связи (ключ → массив строк/ID). |

**Типы отношений** (примеры):
- `надкласс` – наследование (формирует таксономию).
- `разбиение` – классификация на виды.
- `составные части` – меронимия (часть-целое).
- `используется в`, `противопоставляется`, `пояснение`.

>  **Важно**: значения в `relations` могут быть как ID концептов (`C042`), так и строковыми литералами. Механизм `resolve_relations()` корректно обрабатывает оба случая.

##  Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-repo/pioivis-kb.git
cd pioivis-kb
```

### 2. Запуск системы

```bash
python main.py
```

При успешном запуске вы увидите статистику загрузки и главное меню:

```
База знаний загружена: 376 концептов, 12 разделов.

=======================================================
  ИНТЕЛЛЕКТУАЛЬНАЯ ОБУЧАЮЩАЯ СИСТЕМА ПиОИвИС
=======================================================
  Выберите сценарий:

  1 — Изучение теоретического материала
  2 — Глоссарий (поиск термина)
  3 — Интеллектуальный ассистент
  q — Выход
```

##  Тестирование

### Запуск тестов

```bash
python -m unittest discover tests/ -v
```

Тесты покрывают:
- **Валидацию**: формат ID, циклические зависимости, висячие ссылки.
- **Загрузчик**: слияние файлов, индексацию, разрешение связей.
- **Сценарий 1 (Study)** — поиск по разделу, точный и нечёткий поиск термина, форматирование карточки
- **Сценарий 2 (Glossary)** — точный поиск по термину/ID, нечёткий поиск, фильтрация по источнику, разрешение ID в relations
- **Сценарий 3 (Assistant)** — извлечение релевантных концептов, формирование контекста, fallback-поиск, интеграция с Ollama
- **Граничные случаи**: пустые темы, Unicode, длинные термины.

##  Расширение базы знаний

### Добавление нового понятия

1. Откройте соответствующий модуль (например, `glossary_T07.json`).
2. Добавьте новый объект в массив `concepts`:
   ```json
   {
     "concept_id": "C168",
     "term": "мета-онтология",
     "topic_id": "T07",
     "definition": "Онтология, описывающая структуру и семантику других онтологий",
     "relations": {
       "надкласс": ["C042"],
       "пояснение": ["Пример: OWL itself"]
     }
   }
   ```
3. Запустите валидацию: `python -c "from src.kb_loader import KnowledgeBaseLoader; KnowledgeBaseLoader(['data/glossary_T07.json']).load()"`.
4. Перезапустите `main.py`.

### Правила эволюции

- ✅ **Добавление** – всегда разрешено.
- ❌ **Удаление** – запрещено, если на понятие есть ссылки. Вместо удаления добавьте поле `"status": "deprecated"`.
- ❌ **Изменение `concept_id`** – запрещено (нарушит ссылки). Создайте новое понятие и укажите `"replaced_by": "Cxxx"`.
- ✅ **Изменение определения/примеров** – разрешено в любое время.
